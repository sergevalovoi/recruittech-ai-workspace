# /candidate-research — ресёрч кандидата

## Когда использовать
- Перед интервью — чтобы прийти с фактами, а не читать резюме на созвоне
- Перед оффером — финальная проверка
- Проверка рыночных данных (зарплатные вилки, доступность профиля на рынке)

## Почему это работает

Один поиск = поверхностно. Паттерн: несколько параллельных поисков по разным углам → верификация каждого ключевого факта из 2+ источников → синтез. В отчёт попадает только верифицированное; предположения помечаются как предположения.

**Правило AI Mandate №5 действует всегда:** в поиск и в контекст грузим только то, что кандидат сам сделал публичным (профили, публикации, выступления). Внутренние оценки, результаты тестов, персональные данные из ATS — не вставлять в промпты и не выгружать наружу.

## Как запустить

Скажи `/candidate-research` и дай:
- **Кандидат** — имя + текущая компания/роль (или ссылка на публичный профиль)
- **Вакансия** — на какую позицию рассматриваем
- **Глубина** — быстро (5 мин) или детально (15-20 мин)

## Что я сделаю

Параллельные поиски:
1. Публичный профиль (LinkedIn/hh — что доступно без логина)
2. Публикации, статьи, выступления, профессиональные сообщества
3. GitHub / портфолио (если tech-роль)
4. Упоминания компаний из резюме — существуют ли, совпадают ли периоды

Что выясняю:
- **Реальный опыт vs заявленный** — совпадают ли даты, роли, компании
- **Достижения с цифрами** — не «улучшил», а «на сколько %»
- **Паттерн смены работ** — как долго держится, почему уходил
- **Публичная позиция** по профессиональным темам
- **Красные флаги** — расхождения, конфликты, преувеличения

## Формат вывода

```
## [Имя] — [вакансия]
**Вывод: нанять / отказать / уточнить на интервью** + аргументация в 2-3 предложения

### Верифицировано (2+ источника)
- факт [источники]

### Найдено, но не верифицировано
- факт [1 источник, пометка]

### Красные флаги
- ...

### Вопросы на интервью
- 3-5 вопросов, выведенных из находок (проверить расхождения, копнуть достижения)
```
