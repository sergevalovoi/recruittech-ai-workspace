# /eval-screening — оценка резюме против вакансии

## Зачем

Подборка = ~10 резюме в день на вакансию. Читать каждое внимательно — час. Смысл скилла: формальные критерии отсекает автопроверка, соответствие смыслу оценивает агент-судья, ты смотришь только NEEDS_REVIEW (обычно 10-30% подборки). Ты — не масштабируемый ресурс, судья — масштабируемый.

**AI Mandate №5:** в оценку идёт содержимое резюме, которое кандидат сам разослал. Не добавляй в контекст внутренние оценки других людей, зарплатные ожидания из личных переписок и прочее, что кандидат не публиковал.

## Как запустить

Скажи `/eval-screening` и дай:
1. **Вакансию** — текст или файл с требованиями
2. **Резюме** — одно, несколько файлов или папку с подборкой

## Шаг 1 — Критерии (один раз на вакансию)

Из текста вакансии я соберу таблицу критериев и покажу тебе на подтверждение **до** оценки:

| Критерий | Вес |
|---|---|
| [must-have 1: опыт X лет в Y] | HIGH |
| [must-have 2] | HIGH |
| [nice-to-have] | MEDIUM |
| [формат: полнота резюме, контакты] | LOW |

Подтверждённые критерии сохраню в `notes/vacancy-criteria-[название].md` — для следующих подборок по той же вакансии не пересобираем.

## Шаг 2 — Оценка (по каждому кандидату)

Промпт судьи:
```
Ты — строгий судья соответствия кандидата вакансии. Только факты из резюме, не додумывай.

КРИТЕРИИ: [таблица]
РЕЗЮМЕ: [текст]

Вывод:
- Вердикт: PASS / FAIL / NEEDS_REVIEW
- По каждому критерию: соответствует / частично / нет / не указано
- «Не указано» по HIGH-критерию → NEEDS_REVIEW, не FAIL (резюме неполное ≠ кандидат слабый)
- Одна главная причина вердикта, одним предложением
```

## Шаг 3 — Итоговая таблица

```
## Подборка [вакансия] — [дата]
| Кандидат | Вердикт | Ключевая причина |
|---|---|---|

PASS: N — в лонглист
NEEDS_REVIEW: N — посмотри сама, вот что спорно: ...
FAIL: N — причины перечислены, восстановить любого можно вручную
```

Сохраняю в `notes/screening-[вакансия]-[дата].md`.

## Правила

- **FAIL — не приговор.** Всегда показывай причину; человек может вернуть кандидата в лонглист одним словом
- **Судья оценивает соответствие, не «качество человека».** Никаких выводов о личности по резюме
- **Если критерии дают >50% NEEDS_REVIEW** — критерии плохие, пересобираем, а не читаем всё руками
- Перед отправкой лонглиста клиенту — финальный взгляд человека обязателен (10% expert-уровня: граничные кейсы — твои)
