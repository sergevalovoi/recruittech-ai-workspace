# /daily-log — ежедневный чек-ин по метрикам

Это НЕ дневник ради дневника. Это трекер «факт vs план» по твоим рабочим метрикам из `COMPANY.md`. Смысл: разрыв между планом и фактом должен быть виден **каждый день**, а не в конце месяца, когда уже поздно.

## Перед стартом

Прочитай: `COMPANY.md`, `AGENT-PERSONALITY.md`, `metrics-tracker.md` (если есть), последний `notes/daily-log-*.md`.

## Шаг 0 — Первый запуск (если `metrics-tracker.md` не существует)

Возьми метрики и план из `COMPANY.md` (их зафиксировал онбординг). Уточни только то, чего там нет:
1. План на **текущую неделю** по каждой метрике (цифрами)
2. План на **текущий месяц** (цифрами)

Создай `metrics-tracker.md`:
```markdown
# Метрики — план и факт

## Месяц: [YYYY-MM]
| Метрика | План месяц | Факт накопленно | План неделя | Факт неделя |
|---|---|---|---|---|
| [метрика 1] | N | 0 | N | 0 |

*Обновляется через /daily-log и /reflect*
```

Дальше daily-log читает этот файл — не переспрашивай метрики каждый раз.

## Шаг 1 — Мини-интервью (3-5 минут, разговорно, не форма)

1. **Что сделано сегодня по метрикам?** — попроси цифры по каждой активной метрике (собесы/офферы/консультации/выступления — что у человека в трекере). «Ничего по метрике X» — тоже валидный ответ, фиксируй честно
2. **Что двигалось, но в цифру пока не превратилось?** — пайплайн: назначенное, ожидающее ответа
3. **Что мешало?** — блокеры
4. **План на завтра** — 1-2 конкретных пункта, привязанных к метрикам
5. **Энергия:** 🔋 Высокая / ⚡ Средняя / 🪫 Низкая (через AskUserQuestion)

## Шаг 2 — Файл лога

Создай `notes/daily-log-YYYY-MM-DD.md`:
```markdown
# Daily Log — YYYY-MM-DD

## Факт по метрикам сегодня
[метрика: цифра, ...]

## В пайплайне
[что движется]

## Блокеры
[ответ, или "—"]

## Завтра
[1-2 пункта]

## Энергия
[ответ]
```

## Шаг 3 — Обновить `metrics-tracker.md`

Добавь сегодняшний факт к «Факт неделя» и «Факт накопленно». Затем — главное:

**Посчитай и скажи вслух темп:** прошло X% рабочих дней недели/месяца, сделано Y% плана. Если Y заметно меньше X — скажи прямо, без смягчений: «до плана недели не хватает N при таком темпе». Это не упрёк, это навигация. Если темп в норме или выше — тоже скажи, конкретным фактом.

## Шаг 4 — Стрик и короткий отчёт

Стрик = подряд идущие рабочие дни (Пн-Пт) с существующим файлом лога. Не начисляй задним числом и не «восстанавливай» пропуски по просьбе — честный пропуск лучше нарисованного стрика.

Отчёт: 2-4 строки — факт дня, темп vs план, стрик. Если энергия низкая — отметь, не дави. Если был сильный день — назови конкретный факт.

## Edge cases

- **Уже логировал сегодня:** спроси, обновить или пропустить
- **Пропущенный день:** констатируй сброс стрика без нотаций
- **План изменился** (Серж поменял план месяца): обнови `metrics-tracker.md`, старый план не удаляй — допиши историю снизу
- **Три дня подряд факт = 0 по ключевой метрике:** назови это паттерном и спроси, что реально блокирует — это сигнал для разговора с Сержем, не для замалчивания
